This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal blog built on the [tailwind-nextjs-starter-blog](https://github.com/timlrx/tailwind-nextjs-starter-blog) template (v2). Next.js 14 App Router + Contentlayer2 (MDX content pipeline) + Tailwind CSS, written mostly in TypeScript. Content language is zh-CN. Uses `pnpm` (see `packageManager` in package.json).

## Toolchain

This project runs on **Node 20 + pnpm 9.10.0** (the pinned `packageManager`). Use **corepack** to invoke the pinned pnpm — do not rely on a globally self-installed pnpm, since pnpm 11+ requires Node ≥ v22.13 and will crash on Node 20 with `ERR_UNKNOWN_BUILTIN_MODULE: node:sqlite`. Run `corepack enable` once per Node version (nvm installs are per-version); corepack then auto-selects pnpm 9.10.0 from the `packageManager` field.

## Commands

```bash
pnpm dev        # dev server (sets INIT_CWD for contentlayer image sizing)
pnpm build      # next build --no-lint, then scripts/postbuild.mjs (RSS generation)
pnpm serve      # serve the production build
pnpm lint       # eslint --fix across pages/app/components/lib/layouts/scripts
pnpm analyze    # bundle analyzer build (ANALYZE=true)
```

There is no test suite. Lint runs automatically on commit via Husky + lint-staged (eslint + prettier on staged files). `pnpm build` uses `--no-lint`, so run `pnpm lint` separately to catch lint errors.

## Content & the Contentlayer pipeline

This is the core architecture to understand. Content lives in `data/` and is compiled by **Contentlayer2** into typed objects imported from `contentlayer/generated` (aliased to `.contentlayer/generated`). Two document types are defined in `contentlayer.config.ts`:

- **Blog** — `data/blog/**/*.mdx`. Frontmatter: `title` (required), `date` (required), plus optional `tags`, `summary`, `draft`, `images`, `authors`, `layout`, `category`, `lastmod`, `bibliography`, `canonicalUrl`. Computed fields add `slug`, `path`, `readingTime`, `toc`, and `structuredData` (JSON-LD).
- **Author** — `data/author/**/*.mdx`. `data/author/default.mdx` is the default author.

The MDX pipeline wires up remark/rehype plugins for GFM, math (KaTeX), code titles + syntax highlighting (Prism), citations (from `data/references-data.bib`), GitHub-style alerts, and image-to-JSX conversion.

`makeSource`'s `onSuccess` hook runs side effects on every content build:

- Writes tag counts to `app/tag-data.json` (drafts excluded in production). **This file is regenerated by the build — do not hand-edit it** (it currently shows as modified in git because of build runs).
- Optionally writes a local kbar search index to `public/`, and uploads an Algolia index in production. Search provider is selected in `data/siteMetadata.js`.

When you change content frontmatter fields, types, or computed fields, update `contentlayer.config.ts`; the generated types flow through to consuming pages automatically.

## App structure

App Router under `app/`:

- `app/page.tsx` → `app/Main.tsx` — home page.
- `app/blog/[...slug]/page.tsx` — individual posts (catch-all route).
- `app/blog/page/[page]/page.tsx` — paginated blog list.
- `app/tag/[tag]/page.tsx` — posts by tag (uses `app/tag-data.json`).
- `app/projects/`, `app/about/` — static pages (about pulls from `data/author/default.mdx`).
- `app/api/newsletter/route.ts`, `app/api/umami/route.ts` — route handlers.
- `app/seo.tsx`, `app/sitemap.ts`, `app/robots.ts` — SEO/metadata.

**`layouts/`** holds the post and list page templates selected per-post via the `layout` frontmatter field (e.g. `PostLayout`, `PostSimple`, `PostBanner`, `ListLayout`, `ListLayoutWithTags`, `MasonryLayout`, `AboutLayout`). **`components/`** holds shared UI; `components/MDXComponents.tsx` maps MDX tags to React components.

## Configuration

- **`data/siteMetadata.js`** — single source of site config: title, author, URLs, locale, analytics (Umami), comments (Waline), newsletter, and search provider. Change site-wide behavior here.
- **`data/headerNavLinks.ts`**, **`data/projectsData.ts`** — nav and projects content.
- **`next.config.js`** — wraps config with `withContentlayer` and bundle analyzer; defines the Content-Security-Policy (add external service domains to `script-src`/`connect-src` here when integrating analytics, comments, etc.), SVG-as-component webpack rule, and `output: 'standalone'` (or `'export'` when `EXPORT` env set). The Workbox service-worker integration is currently commented out.
- Path aliases (`tsconfig.json`): `@/components/*`, `@/data/*`, `@/layouts/*`, `@/css/*`.

## Scripts

`scripts/postbuild.mjs` (RSS), `scripts/rss.mjs`, `scripts/getAlgoliaObjects.ts` (Algolia index objects), `scripts/service_worker.ts` (Workbox SW source, currently unused).
